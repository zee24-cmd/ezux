# Documentation and Examples Skill

## Scope
Use this skill for README/API docs, examples, prop documentation, migration notes, and consumer-facing guidance.

## Expertise
- React component usage examples, TypeScript snippets, accessibility notes, and enterprise UI documentation.
- Clear API contracts for table, scheduler, kanban, flow, layout, treeview, signature, advanced components, and mock services.

## Rules
- Document public behavior, not internals.
- Keep examples minimal but complete enough to compile.
- Show required peer dependencies and CSS imports when relevant.
- Include controlled and uncontrolled usage where both are supported.
- Call out accessibility, keyboard, timezone/date, virtualization, drag/drop, and sanitization concerns when relevant.
- Keep migration notes concrete: old API, new API, reason, and expected behavior.

## Done Criteria
- Docs match current exports and package entrypoints.
- Examples use current React and TypeScript patterns.
- Any new public prop, event, service, or exported type is documented.
